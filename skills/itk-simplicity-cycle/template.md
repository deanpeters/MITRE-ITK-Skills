## Step 1: Locate Your Project on the Simplicity Cycle

Identify which of the four points best describes your project's current state. Base this on **recent actions and tools** (e.g., if you've been adding lots of features, you're likely near Shift or Stop).

### The Four Phase Points

| Point | State | Description | Recommended Mode |
|-------|-------|-------------|------------------|
| **START** | Simple, basic, immature, delivers little value | [describe how this matches/doesn't match your project] | Additive (increase complexity) |
| **SHIFT** | Critical mass of complexity, delivers significant value | [describe how this matches/doesn't match your project] | Reductive (decrease complexity) |
| **STOP** | Too much complexity, overwhelms value | [describe how this matches/doesn't match your project] | Reductive (significantly reduce) |
| **SHIP** | Elegant: simple AND effective, maximum value | [describe how this matches/doesn't match your project] | Declare complete / release |
